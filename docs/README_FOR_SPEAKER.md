## 🔑 KEY PRINCIPLES

### **Delivery Style: Steve Jobs**
- Stand still, move with purpose
- Make eye contact across the room
- Use pauses instead of filler words
- Tell a story, don't read slides
- Let silence do the work

### **Tone: Honest, Not Salesy**
- "There is no silver bullet"
- "Here's why I picked OpenCode" (honest reasoning)
- "I was frustrated, so I built this"
- Vulnerability builds connection

### **Structure: Problem → Solution → Lessons**
- Don't start with tools
- Start with the gap (problem)
- Show the journey (your experiment)
- Prove it works (Eventyay case study)
- Finish with takeaways

### **Pacing: Slower Than Normal**
- 50 seconds per slide is plenty of time
- Pause 2–3 seconds after big statements
- Let the audience catch up
- Better to finish at 15:30 than rush at 14:00
